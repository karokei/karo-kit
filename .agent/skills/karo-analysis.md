# Karo System Analysis

## Mục đích
Kỹ năng phân tích sâu sắc về mã nguồn, dữ liệu và luồng hoạt động của hệ thống hiện tại để đưa ra các đề xuất cải tiến chính xác.

## Quy trình làm việc
1.  **Thu thập Dữ liệu (Inventory):** Sử dụng `list_dir` để nắm bắt cấu trúc dự án.
2.  **Đào sâu (Deep Dive):** Sử dụng `grep_search` để tìm các logic nghiệp vụ quan trọng hoặc các điểm yếu tiềm tàn.
3.  **Phân tích Mối quan hệ:** Hiểu cách các module tương tác với nhau.
4.  **Đánh giá Hiện trạng:** So sánh với các Best Practices và quy chuẩn của sếp.
5.  **Báo cáo Phân tích:** Trình bày kết quả dưới dạng Markdown chuyên nghiệp với các giải pháp cụ thể.

## Lưu ý cho phân tích
- Luôn giữ cái nhìn khách quan.
- Ưu tiên các vấn đề về hiệu năng và bảo mật.
- Đề xuất các giải pháp có tính khả thi cao.
