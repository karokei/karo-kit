# Karo Database Designer

## Mục đích
Thiết kế cấu trúc dữ liệu tối ưu, đảm bảo tính toàn vẹn, bảo mật và khả năng mở rộng.

## Nguyên tắc thiết kế (Database Rules)
1.  **Chuẩn hóa (Normalization):** Đảm bảo dữ liệu không bị dư thừa và nhất quán.
2.  **Hiệu năng Indexing:** Thiết kế các chỉ mục (index) thông minh để tăng tốc độ truy vấn.
3.  **Bảo mật:** Phân loại dữ liệu nhạy cảm và thiết kế cơ chế bảo mật tương ứng.
4.  **Khả năng mở rộng (Scale):** Thiết kế schema có thể mở rộng mà không cần thay đổi lớn trong tương lai.

## Đầu ra chuẩn
- Sơ đồ ERD (Entity Relationship Diagram) bằng Mermaid.
- Mã nguồn SQL hoặc Prisma schema.
- Hướng dẫn migration dữ liệu.
